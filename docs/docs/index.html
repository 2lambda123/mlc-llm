Welcome to MLC-LLM’s documentation!¶

MLC LLM is a universal solution that allows any language models to be deployed natively on a diverse set of hardware backends and native applications, plus a productive framework for everyone to further optimize model performance for their own use cases.

Our mission is to enable everyone to develop, optimize and deploy AI models natively on everyone’s devices.

Everything runs locally with no server support and accelerated with local GPUs on your phone and laptops.

Join our Discord Server!

Navigation¶

Before you start, please select your use case so that we can narrow down the search scope.

Please select your platform:

Please check our instructions on MLC-LLM Android app.

Please check our instructions on MLC-LLM IOS app

Please check Web-LLM project.

MLC-LLM provided a set of prebuilt models that you can run directly without compile models by your self, check Off-the-Shelf Models and see if your model is in the list.

In other cases, you need you build your own models.

Please check How to Deploy Models for instructions on preparing models and deploying models with MLC-LLM CLI.

  • If you are a Mac OS user, and you will run a model compiled with metal backend, congratulations! You don’t need to install any external drivers/packages but metal is natively supported by Mac OS.

  • If you will run a model compiled with CUDA backend, please install CUDA accordingly to our CUDA installation guide.

  • If you will run a model compiled with Vulkan backend, please install Vulkan Driver accordingly to our Vulkan Driver Installation Guide.

If your model architecture is supported by MLC-LLM, you can compile your own models with MLC-LLM, and customize the model weights/quantization algorithms/running data types/etc. Please check How to Compile Models for details on how to compile models with MLC-LLM.

If the model architecture is not supported yet, you can add a new model architecture to the MLC-LLM family by following tutorial Add New Model Architectures. You can also create a model request issue to request help from the community to add a new model architecture to MLC-LLM family.

In either cases, you are encouraged to contribute to the MLC-LLM, see Contribute New Models to MLC-LLM on guidelines for contributing new models.

There are lots of interesting ways to further improve and customize MLC-LLM.

  • The performance of MLC-LLM can be improved in a lot of ways, including (but not limited to) fusing multi-head attention with FlashAttention algorithm, or using more advanced quantization algorithms.

  • We can also add new backends/language binding with the existing infrastructure.

Before you start, please check our Customize Model Compilation and Optimization to see how can you customize MLC-LLM for your own purpose.

You are encouraged to contribute to the MLC-LLM if your found your customization intersting.

In this case, user need to change TVM-Unity codebase. Please check Option 2. Build from Source on how to install TVM-Unity from source.

  • If user want to compile models with CUDA backend, please install CUDA according to our CUDA installation guide.

  • If user want to compile models with Vulkan backend, please install Vulkan-SDK according to our Vulkan SDK Installation Guide.

  • If user want to compile models with OpenCL backend, please install OpenCL-SDK according to our OpenCL SDK Installation Guide.

In this case, please install prebuilt TVM-Unity package according to our Installation Guidelines.
